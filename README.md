# JEAB Catalog - Behavioral Analysis Database

An interactive web-based platform that reviews every article from the *Journal of the Experimental Analysis of Behavior* (JEAB), logs key behavioral principles, procedural nuances, and translates findings into mathematical formalisms.

## Features

- **Retro Terminal UI**: 1980s science terminal theme with green/amber text on black background
- **Article Database**: Complete JEAB article catalog with metadata
- **Behavioral Principles**: Categorized behavioral analysis principles
- **Mathematical Models**: Static equations and recursive relations with LaTeX rendering
- **Advanced Filtering**: Filter by decade, principle, organism, or keyword search
- **Responsive Design**: Works on desktop and mobile devices

## Architecture

### Backend (FastAPI + PostgreSQL)
- **FastAPI**: Modern Python web framework
- **PostgreSQL**: Railway-hosted database
- **SQLAlchemy**: ORM for database operations
- **Pydantic**: Data validation and serialization

### Frontend (React + Tailwind CSS)
- **React**: Modern JavaScript framework
- **Tailwind CSS**: Utility-first CSS framework
- **KaTeX**: Mathematical equation rendering
- **GitHub Pages**: Static hosting

## Setup Instructions

### Backend Setup

1. **Navigate to backend directory:**
   ```bash
   cd backend
   ```

2. **Create virtual environment:**
   ```bash
   python -m venv venv
   source venv/bin/activate  # On Windows: venv\Scripts\activate
   ```

3. **Install dependencies:**
   ```bash
   pip install -r requirements.txt
   ```

4. **Set up environment variables:**
   Create a `.env` file in the backend directory:
   ```env
   DATABASE_URL=your_railway_postgresql_url
   ```

5. **Run locally:**
   ```bash
   uvicorn main:app --reload
   ```

### Frontend Setup

1. **Navigate to frontend directory:**
   ```bash
   cd frontend
   ```

2. **Install dependencies:**
   ```bash
   npm install
   ```

3. **Set up environment variables:**
   Create a `.env` file in the frontend directory:
   ```env
   REACT_APP_API_URL=your_railway_backend_url
   ```

4. **Run locally:**
   ```bash
   npm start
   ```

## Deployment

### Backend Deployment (Railway)

1. **Create Railway account** at [railway.app](https://railway.app)

2. **Create new project** and connect your GitHub repository

3. **Add PostgreSQL plugin** to your Railway project

4. **Set environment variables:**
   - `DATABASE_URL`: Auto-generated by Railway PostgreSQL plugin

5. **Deploy:**
   Railway will automatically deploy when you push to your main branch

### Frontend Deployment (GitHub Pages)

1. **Update homepage in package.json:**
   ```json
   {
     "homepage": "https://yourusername.github.io/catalog-of-principles-and-processes"
   }
   ```

2. **Deploy to GitHub Pages:**
   ```bash
   npm run deploy
   ```

3. **Configure GitHub Pages** in your repository settings

## API Endpoints

### Articles
- `GET /api/articles` - Get all articles with optional filtering
- `GET /api/articles/{id}` - Get specific article
- `POST /api/articles` - Create new article

### Principles
- `GET /api/principles` - Get all behavioral principles

### Math Models
- `GET /api/math` - Get all mathematical models

### Statistics
- `GET /api/stats` - Get database statistics
